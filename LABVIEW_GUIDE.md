# LabVIEW Integration Guide
## AI-Based Analog Instrument Monitoring System

---

## What LabVIEW needs to do

```
TCP Receive  →  Parse JSON  →  Display  →  Log MySQL  →  Anomaly Alert
```

---

## 1. TCP/IP Receiver VI

### Front Panel Controls & Indicators
| Control/Indicator | Type | Value |
|---|---|---|
| Port | Numeric Constant | 5005 |
| Timeout (ms) | Numeric Constant | 5000 |
| Connection ID | TCP Connection Refnum | (wire) |
| Raw String | String Indicator | (debug) |
| Gauge Value | Numeric Indicator | |
| Timestamp | String Indicator | |

### Block Diagram (pseudocode)

```
[TCP Open Connection]
  host = "" (empty = server mode, wait for client)
  port = 5005
  → connection ID

LOOP (While loop, stop button):

  [TCP Read]
    connection = connection ID
    bytes to read = 256
    mode = CRLF  (read until newline \n)
    → raw string

  [Unflatten from JSON]   ← use LabVIEW JSON VI or Parse JSON String
    raw string → JSON object
    extract: "value"     → DBL wire  → Gauge Value indicator
    extract: "timestamp" → DBL wire  → convert to date/time string

  [Write to Shift Register]
    pass Gauge Value to anomaly subVI (see Section 3)

  [Database Insert]  ← see Section 2

END LOOP

[TCP Close Connection]
```

**TIP:** If your LabVIEW version doesn't have the JSON VI, use the
**Scan From String** function with format `%f` after extracting the
value substring manually.

## 2. MySQL Logging SubVI

### Requirements
- **Database Connectivity Toolkit** must be installed
  (NI Package Manager → NI Database Connectivity Toolkit)

### Block Diagram

```
[DB Tools Open Connection]
  connection string:
  "DSN=GaugeDB;Server=localhost;Database=gauge_monitor;
   UID=root;PWD=yourpassword;"

[Create SQL String]
  format: "INSERT INTO readings (ts, value) VALUES ('%s', %f)"
  wire in: timestamp_string, gauge_value_dbl

[DB Tools Execute Query]
  connection reference + SQL string

[DB Tools Close Connection]
```

### MySQL Setup (run once in MySQL Workbench or terminal)

```sql
CREATE DATABASE IF NOT EXISTS gauge_monitor;
USE gauge_monitor;

CREATE TABLE IF NOT EXISTS readings (
    id        INT AUTO_INCREMENT PRIMARY KEY,
    ts        DATETIME NOT NULL,
    value     DOUBLE NOT NULL,
    anomaly   TINYINT DEFAULT 0
);
```

## 3. Anomaly Detection SubVI

### Algorithm: Rolling Mean ± 3σ

```
Inputs:
  new_value   (DBL)
  history[]   (DBL array, shift register, size=30)

Block Diagram:

  [Array Max & Min + Mean]   ← use Array Statistics VI
    history → mean, std_dev

  upper = mean + 3 * std_dev
  lower = mean - 3 * std_dev

  anomaly? = (new_value > upper) OR (new_value < lower)

  [Insert Into Array]
    append new_value to history, keep last 30

Outputs:
  mean        (DBL)
  upper       (DBL)
  lower       (DBL)
  anomaly?    (Boolean) → LED indicator on Front Panel
  history[]   (DBL array, back to shift register)
```

### Front Panel
- **Waveform Chart** — plots Value, Mean, Upper, Lower as 4 plots
- **Boolean LED** — red when anomaly = TRUE
- **Numeric** — shows current value large

## 4. Full VI Architecture

```
Main.vi
├── TCPReceiver.vi       (runs in loop, outputs value + timestamp)
├── AnomalyDetector.vi   (subVI, shift-register history)
├── MySQLLogger.vi       (subVI, called each iteration)
└── Front Panel
    ├── Waveform Chart (4-plot: value, mean, +3σ, -3σ)
    ├── Numeric Indicator (big, current value)
    ├── Boolean LED (red = anomaly)
    ├── String (last anomaly timestamp)
    └── Stop Button
```
